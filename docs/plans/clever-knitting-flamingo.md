# Code Checker 前端重构计划

## 背景

Code Checker 当前采用类似 ChatGPT 的三栏对话式布局，但该产品实际上是结构化的代码审查工具，包含固定的分析流程：
1. 提交代码和问题描述
2. AI 多阶段分析（QUEUED → PROCESSING → COMPLETED）
3. 展示五部分结构化结果：原始提交、问题详情、源代码、代码对比、分析建议

ChatGPT 风格适合开放式对话，但不适合这种结构化的代码审查场景。

## 目标

重构为 GitHub PR 风格的代码审查界面：
- 以代码对比为核心，并排展示原始代码和修复后代码
- 分层信息架构，而非 Tab 切换
- 保持现有 UI 设计系统的约束（极简、12个变量、边框优先）

## 设计方案

### 布局架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│  [返回]  Code Checker    分析 #42    [状态Badge]    [重试] [关闭]        │  ← PageHeader
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌─────────────────────────────┬─────────────────────────────┐         │
│  │  📄 original_code.py        │  📄 fixed_code.py           │         │  ← 核心代码对比区
│  │  ─────────────────────────  │  ─────────────────────────  │         │     (始终可见)
│  │                             │                             │         │
│  │  [语法高亮的并排代码对比]      │  [语法高亮的并排代码对比]      │         │
│  │                             │                             │         │
│  │                             │                             │         │
│  └─────────────────────────────┴─────────────────────────────┘         │
│                                                                         │
│  ▼ 原始提交                                    [展开/收起]             │  ← 折叠面板
│  ─────────────────────────────────────────────────────────────────     │
│  用户输入的问题描述和截图                                               │
│                                                                         │
│  ▼ 问题详情                                                          │
│  ─────────────────────────────────────────────────────────────────     │
│  AI提取的问题标题、描述、严重程度、标签                                  │
│                                                                         │
│  ▼ 逐块分析                                                          │
│  ─────────────────────────────────────────────────────────────────     │
│  每个修改点的详细说明（位置、修改前后、理由）                            │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 关键变更

1. **移除 Tabs**：RequestDetail 当前使用 Tabs 切换不同内容，改为并排代码对比 + 折叠面板
2. **新增并排 Diff**：使用 diff2html 的 side-by-side 模式作为主要内容区
3. **折叠面板组件**：创建可展开/收起的 Section 组件展示元信息
4. **保持侧边栏**：RequestList 侧边栏保留，用于快速切换历史记录

### 文件变更

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `src/components/RequestDetail.tsx` | 大幅修改 | 移除 Tabs，改为新布局 |
| `src/components/Dashboard.tsx` | 小幅修改 | 调整布局比例，给代码对比更多空间 |
| `src/components/ui/collapsible-section.tsx` | 新增 | 折叠面板组件 |
| `src/app/globals.css` | 小幅修改 | 添加代码对比区所需的样式覆盖 |

### 技术实现

1. **代码对比**：复用现有 diff2html，但改为双栏并排展示（当前代码已有此功能，只是需要调整布局位置）
2. **折叠面板**：使用 Radix UI 的 Collapsible 组件，保持可访问性
3. **响应式**：桌面端并排展示，移动端改为上下堆叠或保持 Tab 切换
4. **动效**：遵循 UI.md 约束，仅使用淡入和 ≤2px 位移

### 验证方式

1. 启动开发服务器：`npm run dev`
2. 访问已有分析结果页面
3. 验证：
   - 并排代码对比是否正确显示
   - 折叠面板能否正常展开/收起
   - 侧边栏历史切换是否正常
   - 移动端布局是否正常

## 风险

1. **代码对比渲染性能**：大文件 diff 可能影响性能（现有代码已处理，保持不变）
2. **移动端体验**：并排对比在小屏幕可能不理想（方案：移动端回退到 Tab 或上下堆叠）
